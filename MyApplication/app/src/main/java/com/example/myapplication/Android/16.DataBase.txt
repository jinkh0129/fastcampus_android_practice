

16. 데이터 베이스
 - 데이터를 저장하기 위한 공간
 - 데이터 저장방식
    - RDB(Relational DataBase) : 관계형 데이터 베이스
     --> 엑셀과 유사한 형식(행과 열로 구성된 표)
     --> 가장 많이 사용되는 방식

    -Key-Value 방식
     --> 쌍으로 저장하는 방식(키와 밸류)

    - RDBMS(Relational DataBase Management System) : RDB를 관리하기 위한 툴
     --> MySQL
     --> Oracle
     --> PostgreSQL

    - SQL
     --> RDBMS를 위한 언어이다.
    - NoSQL
     --> SQL말고 다르게 저장을 해보자!라는 식에서 나온 것
     --> 그 중에서 가장 성공한 것이 키밸류 방식
        - 키밸류 방식의 장점은 검색속도가 빠르다는 것이다.
        - 검색엔진을 만들 때는 키밸류 방식이 빠르다.


    - 안드로이드 데이터베이스
        --> SQLiteDatabase
        --> SharedPreference
            - 목적 : 공유된 사용자의 기호(사용자의 앱 설정 등)를 저장하기 위한 데이터베이스
            - Key_Value방식
            - 간단한 것들을 저장하기 위한 용도
            - 하드하게 데이터베이스 시스템을 구축할 수 없다.
            - 예) 사용자가 처음 어플에 들어왔을 때만 로그인을 하고 나중에 들어올 때는 로그인 정보가 저장이 되어서
              다시 로그인을 하지 않아도 되게 한다.
